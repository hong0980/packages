XSS保护

此仪表板将要渲染来自服务器的数据：

为了保护您的隐私，信息中心将检查所有传输的数据以进行跨站点脚本（XSS）。
这会占用大量CPU，因此您的浏览器可能会慢一些。

如果您信任远程服务器，则可以禁用XSS保护。
在这种情况下，任何远程仪表板装饰代码（javascript）也将运行。

如果您不信任远程服务器，则应保持保护状态。
仪表板将运行速度较慢，并且远程仪表板装饰代码将不会运行，但是比后悔更好安全...

鼠标移至/悬停

将鼠标悬停在图表上可以在其图例处显示鼠标下的时间戳记值（图表还将在图表上突出显示该点）。
所有其他可见图表还将显示并突出显示相同时间戳的值。

拖动图表内容

通过按下鼠标左键并移动鼠标，拖动图表的内容以水平平移。
您放开图表后，所有图表都会很快出现（这种小延迟是设计使然：它可以加快浏览器的速度，并让您专注于正在探索的内容）。
平移图表后，将停止所有图表的自动刷新。要再次启用它，请双击窗格中的图表。

双击

双击图表可将所有图表重置为其默认的自动刷新状态。

SHIFT + 拖动

在按下 SHIFT 键的同时，在图表的内容上按鼠标左键，然后移动鼠标以选择要放大的区域。其他图表也将随之出现。 缩放分为两个阶段：
  • 已加载的图表内容被缩放（低分辨率）
  • 从netdata服务器传输新数据，以刷新图表的更多细节。
图表缩放后，所有图表将停止自动刷新。要再次启用它，请双击缩放的图表。

突出显示时间

在按 ALT 键的同时，在图表内容上按鼠标左键，然后移动鼠标选择一个区域。所选的将在所有图表上突出显示。

SHIFT +鼠标滚轮

在按 SHIFT 键并且鼠标指针悬停在图表内容上方时，滚动鼠标滚轮可以放大或缩小。这种缩放将对齐到鼠标指针下方的中心。其他图表也将遵循。
图表缩放后，所有图表将停止自动刷新。要再次启用它，请双击缩放的图表。

图例操作

单击尺寸的标签或值，将选择/取消选择该尺寸。
您可以按SHIFT或CONTROL键中的任何一个，然后单击图例标签或值，以选择/取消选择多个尺寸。

单击

单击图表内容可在其图例处显示所敲击的时间戳记的值（该图表还将突出显示该图表上的点）。
所有其他可见图表还将显示并突出显示相同时间戳的值。

拖动图表内容

触摸并拖动图表的内容以将其水平平移。
您放开图表后，所有图表都会很快出现（这种小延迟是设计使然：它可以加快浏览器的速度，并让您专注于正在探索的内容）。
平移图表后，将停止所有图表的自动刷新。要再次启用它，请双击一个平移的图表。

双击

双击图表将所有图表重置为其默认的自动刷新状态。

缩放 （不适用于Firefox和IE/Edge）

用两根手指放大或缩小。
图表缩放后，所有图表将停止自动刷新。要再次启用它，请双击缩放的图表。

图例操作

点击尺寸的标签或值，将选择/取消选择该尺寸。

什么时候刷新图表？
当设置为聚焦时，如果页面/选项卡没有引起用户的关注，则图表将停止更新。当设置为始终时，图表将始终刷新。将其设置为聚焦时，如果该页面没有聚焦，则可降低浏览器的CPU要求（并延长笔记本电脑和平板电脑的电量）。设置为始终可以在另一个窗口上工作（即更改某些设置），并在该窗口中自动刷新图表。
要显示哪个尺寸？
设置为非零时，所有值（在当前视图内）都设置为零的维将不会从netdata服务器传输（除非图表的所有维均为零，在这种情况下，此设置不执行任何操作，所有尺寸均已传输并显示）。设置为全部时，将始终显示所有尺寸。将其设置为非零可以降低在netdata和浏览器之间传输的数据，降低浏览器的CPU要求（减少绘制线条）并增加对图例的关注（图例中的条目更少）。
如何处理隐藏图表？
设置为消去时，不在浏览器当前视口中（在页面可见区域的上方或下方）的统计图将被消去，并在以下情况下重新创建他们再次变得可见。当设置为隐藏时，不可见的图表将被隐藏，以简化DOM并加快浏览器的速度。将其设置为消去，以降低浏览器的内存需求。将其设置为隐藏可以更快地恢复页面滚动时的图表。
页面滚动处理？
设置为同步时，将在滚动后立即检查图表的可见性。在速度较慢的计算机上，这可能会影响页面滚动的流畅性。要在滚动结束时更新页面，请将其设置为异步。将其设置为同步，以在滚动时立即更新图表。将其设置为异步可以在速度较慢的计算机上更流畅地滚动页面。
采用何种图表更新模式？
设置为并行时，并行刷新可见图表（所有查询并行发送到netdata服务器）并异步呈现。设置为顺序时，图表会一次又一次刷新。 如果您的浏览器可以应对，请将其设置为并行（大多数现代浏览器都可以），如果您使用的计算机较旧/较慢，则将其设置为顺序。
重新整理时是否需要同步所有图表？
设置为重新同步时，仪表板将尝试重新同步所有图表，以便同时刷新它们。设置为尽力而为时，每张图表的刷新时间可能与其他图表略有不同。通常仪表板会开始并行刷新它们，但是根据您计算机的速度和网络延迟，图表开始会有细微的时差。将此设置为重新同步将尝试在每次更新时重新同步图表。将其设置为尽力而为可能会减轻浏览器和网络的压力。
是否悬停在图表上时同步更新所有图表？
启用后，一个图表上的选择将自动在所有其他可见图表上同时选择，并且所有可见图表的图例将更新以显示所选值。禁用后将仅选择引起用户注意的图表。启用它可以更好地了解数据。如果您使用的计算机速度非常慢而实际上无法执行此操作，请禁用它。
是否同步移动与缩放所有图表？
启用后图表上的平移和缩放操作将复制到所有图表（甚至是当前不可见的图表。当然，只有在它们变为可见时）。禁用后平移和缩放操作将不会传播到其他图表。
使用那一种主题？
Netdata有两个主题：深色（默认）和浅色。
切换此设置将重新加载仪表板。
您需要说明吗？
Netdata可以在某些方面显示一些帮助，以帮助您使用仪表板。如果所有这些气球都困扰您，请使用此开关将其禁用。
切换此设置将重新加载仪表板。
是否启用在移动与缩放时同时填充数据？
设置为填充时，图表将在可见区域之前和之后填充更多数据，从而给整个数据库加载一个印象。仅在图表上的第一个平移或缩放操作之后才发生填充（最初所有图表仅具有可见数据）。设置为不填充时，即使在第一次平移和缩放操作之后，也只会从可见数据中传输数据。
是否在图表上启用贝兹曲线？
设置为平滑时，支持该功能的图表库将绘制平滑曲线而不是简单的直线来连接点。
请记住，NetData仪表板中的主要图表库 图表只能平滑折线图。它无法平滑区域或堆积的图表。设置为粗糙时，此设置可以降低浏览器消耗的CPU资源。
随着图表的更新，这些设置将逐渐应用。要强制使用它们，请立即刷新仪表板。
是否启用自动缩放单位选择？
设置为自动缩放单位时，显示的值将动态缩放（例如1000千位将显示为1兆位）。 Netdata可以自动缩放以下原始单位：kilobits/s, kilobytes/s, KB/s, KB, MB, 和 GB. 设置为固定单位时，所有值将使用netdata服务器维护的原始单位进行呈现。
使用何种温度单位？
设置仪表盘的温度单位。
是否将时间转换为秒？
当设置为时间时，显示秒的图表将显示DD:HH:MM:SS。 当设置为秒时，将显示原始秒数。
显示浏览器本地时间还是服务器时间？
设置为本地时间时，图表将使用您的浏览器本地时间。设置为服务器时间时，图表将使用服务器时间。
Set it to 本地时间无论服务器位于何处以及使用哪个时区，都有一个通用的时间参考（所有仪表板都会报告您的本地时间）。 Set it to 服务器时间需要将NetData图表与服务器日志文件进行比较时。
